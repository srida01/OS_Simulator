INTERRUPT FLOW

The operating system handles interrupts through a precise sequence of actions. The CPU must preserve the currently running process, execute privileged kernel code, handle the event, and safely restore execution.

STEP 01
CPU Executes Current Process
The processor fetches and executes instructions from the currently running program.
STEP 02
Interrupt Occurs
A hardware device or software event sends an interrupt request to the processor.
STEP 03
Save Context
The CPU saves registers, program counter, and execution state before switching tasks.
STEP 04
Switch to Kernel Mode
Privileged operating system code gains control to safely handle the interrupt.
STEP 05
Locate Interrupt Handler
The interrupt vector table provides the memory address of the correct ISR.
STEP 06
Execute ISR
The operating system processes the interrupt event and performs the required service.
STEP 07
Restore Execution
Saved process information is restored and execution resumes from the exact interruption point.
