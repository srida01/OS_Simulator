Theory

Single Buffer

One shared buffer. Producer fills it, then waits for the consumer to empty it. Simple but inefficient — both agents cannot work simultaneously.

Double Buffer

Two buffers alternate roles. Producer fills Buf A while consumer drains Buf B, then they swap. Eliminates wait time, enabling true parallel operation.

Circular Buffer

A ring of N slots with write (head) and read (tail) pointers. Full when head+1 = tail; empty when head = tail. Efficient for streaming I/O pipelines.

Why Buffering?

I/O devices are orders of magnitude slower than the CPU. Buffering decouples them, absorbing speed mismatches and improving overall system throughput.

How DMA Works

Phase 1 — Programming

CPU writes the source address, destination address, byte count, and transfer direction into the DMA controller's registers via I/O instructions.

Phase 2 — Bus Arbitration

DMA controller asserts the HOLD signal. CPU finishes its current bus cycle, then asserts HLDA (Hold Acknowledge) and releases the system bus.

Phase 3 — Data Transfer

DMA acts as bus master, reading from the I/O device and writing directly to main memory — entirely without CPU involvement. Throughput is maximized.

Phase 4 — Interrupt

When transfer completes, DMA fires an IRQ. CPU regains bus control, handles the interrupt, and resumes normal execution with data now in RAM.

HOW TO USE

Buffering Simulator

  • Choose Single, Double, or Circular buffer mode using the selector.
  • Adjust buffer size (4–12 slots) and simulation speed with the sliders.
  • Press Run for continuous simulation or Step for one operation at a time.
  • Blue cells are producer writes; green flashes are consumer reads. Amber = waiting.
  • In Double mode, watch both buffers work simultaneously — no idle time!
  • In Circular mode, W = write head, R = read tail. The ring wraps automatically.

DMA Simulator

  • Set the number of data blocks to transfer and the playback speed.
  • Press Start DMA to run through all phases automatically.
  • Use Step to advance one DMA phase at a time and read each event in detail.
  • Notice the CPU turns SUSPENDED during the actual transfer — it's free for other work.
  • The progress bar shows real-time transfer completion during Phase 3.
