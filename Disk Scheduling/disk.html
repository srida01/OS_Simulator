Disk Scheduling

Disk scheduling is the technique used by operating systems to determine the sequence in which block I/O operations are dispatched to storage volumes. Also known as I/O scheduling, it orchestrates the order of disk operations to optimize performance and resource utilization.

Key Terms

// important concepts related to disk scheduling algorithms

Seek Time
The time taken by the disk arm to locate the required cylinder on disk. The algorithm minimising seek time is generally considered more efficient.
Rotational Latency
Time for the required disk block to rotate under the read/write head. Minimising this contributes directly to lower access time.
Transfer Time
Time taken to actually transfer data between the disk and main memory once the head is positioned correctly.
Disk Access Time
Total time = Seek Time + Rotational Latency + Transfer Time. The primary metric used to compare scheduling algorithms.
Disk Response Time
Average waiting time of all requests in the queue. Lower response time indicates a fairer, more responsive scheduling policy.

Scheduling Algorithms

// select an algorithm below and visualise its behaviour interactively

FCFS Basic
First Come First Serve
Requests are served in the order they arrive. Simple to implement but does not optimise head movement, leading to high seek times under heavy load.
Learn more
SSTF Greedy
Shortest Seek Time First
Prioritises the request closest to the current head position. Reduces average seek time compared to FCFS but may cause starvation for far requests.
Learn more
SCAN Sweep
Elevator Algorithm
The arm sweeps in one direction servicing all requests, then reverses. Behaviour mirrors an elevator — hence the nickname.
Learn more
C-SCAN Circular
Circular SCAN
Like SCAN, but after reaching the last cylinder the head jumps back to the first without servicing any requests, then sweeps again for uniform wait times.
Learn more
LOOK Optimised
Modified SCAN
Like SCAN but the arm reverses direction as soon as there are no more requests ahead, avoiding unnecessary travel to the disk edge.
Learn more
C-LOOK Circular
Circular LOOK
The arm services requests outward until the highest request, then jumps to the lowest without servicing, combining LOOK efficiency with C-SCAN fairness.
Learn more
LIFO Stack
Last In First Out
The most recently arrived request is serviced first. Useful for certain cache-locality scenarios but can cause starvation of older requests.
Learn more
RSS Stochastic
Random Seek Scheduling
Picks the next request at random. Used in simulation and analysis to model stochastic environments and establish baseline comparisons.
