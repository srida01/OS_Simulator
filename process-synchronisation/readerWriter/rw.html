READER-WRITER
PROBLEM

Module 12 · Synchronisation
Overview

A shared object (file or record) is accessed by two classes of processes:

  • Readers — only read the shared object
  • Writers — read and write the shared object

Simultaneous access by a writer and any other process causes data inconsistency.

Variations

First R-W Problem

No reader waits unless a writer has already obtained permission.

  • Readers proceed even if writers are waiting.
  • ⚠ Writers may starve.

Second R-W Problem

Once a writer is ready, it writes as soon as possible. New readers are blocked.

  • ⚠ Readers may starve.
Implementation — First R-W Problem

Shared Variables:

  • readcount = 0 — active reader count
  • mutex = 1 — guards readcount
  • wrt = 1 — writer mutual exclusion

Writer

wait(wrt); // writing is performed signal(wrt);

Reader

wait(mutex); readcount++; if (readcount == 1) wait(wrt); signal(mutex); // reading is performed wait(mutex); readcount--; if (readcount == 0) signal(wrt); signal(mutex);
