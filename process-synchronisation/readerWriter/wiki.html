Readers-Writers Problem

Shared Data: (e.g., a file or record)

  • Readers – only read the shared object
  • Writers – update (read and write) the shared object

Problem: If a writer and another process (reader or writer) access the shared data simultaneously, it can lead to inconsistency.

Variations of the Readers-Writers Problem

  1. First Readers-Writers Problem:
    No reader should wait unless a writer has already obtained permission to use the shared object.
    • Readers do not wait if a writer is only waiting.
    • Writers may starve.
  2. Second Readers-Writers Problem:
    Once a writer is ready, it should write as soon as possible, preventing new readers from starting.
    • Readers may starve.

First Readers-Writers Problem Implementation

Shared Variables:

  • int readcount – keeps track of how many readers are currently accessing the object
  • semaphore mutex – ensures mutual exclusion when readcount is updated
  • semaphore wrt – mutual exclusion for writers

Initial Values:

  • mutex = 1
  • wrt = 1
  • readcount = 0
Writer Process
wait(wrt);
// ...
// writing is performed
// ...
signal(wrt);
Reader Process
wait(mutex);
readcount++;
if (readcount == 1)
    wait(wrt);
signal(mutex);

// ...
// reading is performed
// ...

wait(mutex);
readcount--;
if (readcount == 0)
    signal(wrt);
signal(mutex);
