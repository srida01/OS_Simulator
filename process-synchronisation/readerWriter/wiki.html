READERS-WRITERS
PROBLEM

Module 12 · PROCESS SYNCRONISATION — Reference
Overview

Shared Data (e.g. a file or record) is accessed concurrently by two classes of processes:

  • Readers — only read the shared object; do not modify it.
  • Writers — update (read and write) the shared object.

Problem: If a writer and any other process access the shared data simultaneously, it can lead to data inconsistency.

Variations

First Readers-Writers Problem

No reader should wait unless a writer has already obtained permission to use the shared object. Readers do not wait even if a writer is queued.

⚠ Writers may starve

Second Readers-Writers Problem

Once a writer is ready, it writes as soon as possible. New readers are blocked until the writer finishes.

⚠ Readers may starve
Shared Variables — First R-W Problem
int readcount
= 0
Tracks how many readers are currently accessing the object.
semaphore mutex
= 1
Ensures mutual exclusion when readcount is updated.
semaphore wrt
= 1
Provides mutual exclusion for writers (and first/last reader).
Writer Process
wait(wrt); // writing is performed signal(wrt);
Reader Process
wait(mutex); readcount++; if (readcount == 1) wait(wrt); signal(mutex); // reading is performed wait(mutex); readcount--; if (readcount == 0) signal(wrt); signal(mutex);
