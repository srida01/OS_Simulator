Dining Philosophers Problem

The Dining Philosophers Problem illustrates the challenges of resource allocation and process synchronization. It involves five philosophers who alternate between thinking and eating. Each needs two chopsticks (one on the left and one on the right) to eat from a shared bowl of rice.

Problem Description

  • Five philosophers sit around a table.
  • Each has one chopstick on their left and right.
  • They can pick up one chopstick at a time.
  • They eat only when both chopsticks are acquired.
  • After eating, they release the chopsticks and go back to thinking.

Goal

Design a solution that allows concurrent access while avoiding deadlock and starvation.

Shared Data
  • semaphore chopstick[5] – represents each chopstick (all initialized to 1).
Philosopher Process (Pseudocode)
do {
    wait(chopstick[i]);
    wait(chopstick[(i+1)%5]);
    // ...
    // eat
    // ...
    signal(chopstick[i]);
    signal(chopstick[(i+1)%5]);
    // ...
    // think
    // ...
} while (TRUE);

Deadlock Issue

If all philosophers pick up their left chopstick at the same time, none can acquire the right one, resulting in a deadlock. All semaphores are then set to 0, and the system halts.

Deadlock Handling Strategies

  • Limit Concurrency: Allow at most four philosophers to sit at the table simultaneously.
  • Check Availability: Allow picking up chopsticks only if both are available (in a critical section).
  • Asymmetric Solution: Odd-numbered philosophers pick up the left chopstick first, while even-numbered ones pick up the right first to avoid circular wait.
