Sleeping Barber Problem

The Sleeping Barber Problem is a classic synchronization problem that illustrates the challenges of coordinating multiple processes (customers and a barber) with limited shared resources (chairs).

Scenario: A barber sleeps when no customers are present. If a customer arrives and the barber is asleep, the customer wakes him up. If the barber is busy and all chairs are occupied, the customer leaves. Otherwise, the customer waits.

Key Concepts:

  • Waiting Chairs: Limited number of seats in the waiting room.
  • Barber Chair: Only one customer can be serviced at a time.
  • Customers: Arrive at random intervals seeking a haircut.
  • Barber: Sleeps if no customer is waiting.

Synchronization Tools:

  • semaphore waitingChairs – controls access to limited chairs.
  • semaphore barberReady – signals barber availability.
  • semaphore accessSeats – mutex to safely update chair count.
Customer Process
wait(accessSeats);
if (availableChairs > 0) {
    availableChairs--;
    signal(barberReady);  // wake up the barber
    signal(accessSeats);
    wait(barberChair);    // wait for barber to be ready
    // get haircut
} else {
    signal(accessSeats);
    // leave the shop
}
Barber Process
while (true) {
    wait(barberReady);     // sleep until customer arrives
    wait(accessSeats);
    availableChairs++;
    signal(accessSeats);
    signal(barberChair);   // ready for haircut
    // cut hair
}
