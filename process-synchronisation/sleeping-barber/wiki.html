SLEEPING BARBER

Synchronization · Semaphores

The Sleeping Barber is a classic inter-process synchronization problem that models the challenge of coordinating concurrent processes with limited shared resources.

THE SCENARIO

A barber shop has one barber, one barber chair, and a waiting room with a fixed number of chairs. When no customers are present the barber sleeps. A new customer either wakes the barber if he is asleep, waits in the waiting room if the barber is busy and a chair is free, or leaves immediately if all chairs are occupied.

This maps directly to producer-consumer problems in OS design: the barber is the consumer, customers are producers, and the waiting room chairs are the bounded buffer.

KEY CONCEPTS
Resource · Bounded Buffer

WAITING CHAIRS

A fixed number of seats in the waiting room that acts as the bounded buffer between arriving customers and the barber.

Resource · Critical Section

BARBER CHAIR

Only one customer can be serviced at a time. Access to the barber chair is the critical section protected by a semaphore.

Process · Producer

CUSTOMERS

Arrive asynchronously seeking service. Their behavior depends on the current state of the waiting room and barber.

Process · Consumer

BARBER

Sleeps when no work is available. Woken by a semaphore signal when a customer arrives. Processes customers sequentially.

SYNCHRONIZATION TOOLS
  • semaphore waitingChairs — Controls access to the limited waiting chairs; prevents more customers from entering than there are seats.
  • semaphore barberReady — Signals barber availability. A customer signals this to wake the barber; the barber waits on it when idle.
  • semaphore accessSeats — Mutex that serializes reads/writes to the shared availableChairs counter, preventing race conditions.
CUSTOMER PROCESS
customer.pseudo 
wait(accessSeats);
if (availableChairs > 0) {
    availableChairs--;
    signal(barberReady);   // wake barber
    signal(accessSeats);
    wait(barberChair);     // wait for service
    // get haircut
} else {
    signal(accessSeats);
    // leave — no chairs available
}
BARBER PROCESS
barber.pseudo 
while (true) {
    wait(barberReady);    // sleep until customer arrives
    wait(accessSeats);
    availableChairs++;
    signal(accessSeats);
    signal(barberChair);  // ready for next customer
    // cut hair
}
