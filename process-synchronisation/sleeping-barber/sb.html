SLEEPING BARBER

A classic synchronization problem illustrating how limited resources (chairs) coordinate concurrent processes (customers & barber) via semaphores.

simulation · input
Chair count must be greater than 0.
pseudocode · barber & customer
Customer Process
wait(accessSeats);
if (availableChairs > 0) {
availableChairs--;
signal(barberReady);
signal(accessSeats);
wait(barberChair);
} else {
signal(accessSeats); // leave
}

Barber Process
while (true) {
wait(barberReady);
availableChairs++;
signal(barberChair);
// cut hair
}
