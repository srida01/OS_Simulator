Problem Description

The Producer-Consumer problem is a classic example of a multi-process synchronization scenario. It involves two types of processes: producers, which generate data and place it into a buffer, and consumers, which remove data from the buffer and process it.

Key Concepts

  • A shared buffer pool with n slots (each holding one item).
  • Producers add items to the buffer.
  • Consumers remove items from the buffer.
  • Synchronization is needed to prevent race conditions and ensure buffer availability.

Shared Data (Semaphores)

  • semaphore mutex – ensures mutual exclusion while accessing the buffer.
  • semaphore full – counts the number of full buffers.
  • semaphore empty – counts the number of empty buffers.

Initial values: mutex = 1, full = 0, empty = n

Producer Process

do {
  // produce an item in nextp
  wait(empty);
  wait(mutex);
  // add nextp to buffer
  signal(mutex);
  signal(full);
} while (1);

Consumer Process

do {
  wait(full);
  wait(mutex);
  // remove an item from buffer to nextc
  signal(mutex);
  signal(empty);
  // consume the item in nextc
} while (1);

Note: There is symmetry between the producer and consumer processes in how they acquire and release semaphores.
