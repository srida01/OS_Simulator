RESOURCE ALLOCATION GRAPHS

A directed graph used to detect deadlocks in operating systems — visually representing relationships between processes and resources.

Graph Elements

  • Processes — represented as circles
  • Resources — represented as squares
  • Request Edge — Process → Resource (wants to acquire)
  • Assignment Edge — Resource → Process (already allocated)

Deadlock Detection

A deadlock exists if and only if the RAG contains a cycle. When detected, a set of processes are each waiting for resources held by another — creating a circular wait condition.
